# 前端页面能力规划

## Capability
将当前单页式后台拆分为多页面工作台，让不同任务在独立页面内完成，降低认知负担，符合常规企业后台的使用习惯。

## 页面划分

### 1. 总览页 `/`
- 面向所有已登录用户
- 负责展示系统定位、当前账号可见范围、常用入口、操作建议
- 不承载复杂表单和长流程操作

### 2. 问答页 `/qa`
- 面向所有已登录用户
- 负责提问、查看路由结果、查看 citations、会话历史与会话管理
- 不承载文档上传和用户授权表单

### 3. 知识运营页 `/knowledge`
- 主要面向管理员
- 负责上传文档、覆盖索引、查看最近上传记录、查看知识运营建议
- 普通用户可以看到页面，但不应拥有上传能力

### 4. 权限管理页 `/users`
- 仅管理员可执行核心操作
- 负责创建用户、角色分配、来源授权、重置密码、删除用户
- 普通用户进入后只能看到说明，不应拥有管理能力

## 固定约束
- 登录页和注册页继续保持独立页面
- 非登录用户访问后台页面时，应跳转到登录页
- 页面之间保持统一后台壳：品牌、导航、当前账号、退出登录、修改密码
- 能力边界按页面隔离，不再把上传、提问、授权全部塞入同一页面

## 非目标
- 这一轮不做新的业务功能接口
- 不改动后端权限模型
- 不引入新的前端框架

## 交付判断
- 用户可以从总览页清楚知道去哪里提问、去哪里上传、去哪里授权
- 问答、知识运营、权限管理三类任务不再出现在同一工作页面
- 现有登录、问答、上传、权限接口仍可用
